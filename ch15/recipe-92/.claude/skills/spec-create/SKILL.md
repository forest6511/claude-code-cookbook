# Feature Specification

Create a spec for: $ARGUMENTS

Save to specs/{feature-name}.md with these
sections:

1. Overview (1-2 sentences)
2. Requirements (checklist format)
3. Non-functional requirements
4. API design (endpoints, input/output)
5. Data model
6. Test plan (test case checklist)
7. Implementation notes
